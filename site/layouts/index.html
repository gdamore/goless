Why goless

Standalone And Embeddable

Useful as a text viewer in its own right, but built so host applications can integrate the same core without shelling out.

Modern

Supports 24-bit color, OSC 8 links, real Unicode grapheme handling, and cleaner chrome than classic terminal pagers.

Intuitive

Familiar enough to pick up quickly, but designed to feel clearer and less awkward than inherited 1980s pager behavior.

Powerful

Search modes, line numbers, follow mode, scrolling, headers, and presentation controls are available when you need them.

Secure

Treats input as data, sanitizes unsupported terminal sequences, and makes hyperlink activation an explicit policy choice.

Customizable

Themes, chrome, prompts, key bindings, render modes, and hyperlink decisions stay under embedder control.

Technical Fit

Pure Go

Implemented in pure Go, without a cgo dependency in the pager core, so builds and cross-compilation stay straightforward.

tcell-Native

Designed to render onto tcell.Screen, which makes it a natural fit for applications already using the Go terminal UI ecosystem.

Focused Core

Built as a pager and viewer core rather than a terminal emulator, keeping the integration story tighter and the behavior easier to reason about.
